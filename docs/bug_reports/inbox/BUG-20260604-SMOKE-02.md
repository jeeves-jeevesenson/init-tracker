# BUG-20260604-SMOKE-02

## Title
Browser Smoke Harness lacks support for `composite` action types (Multiattack)

## Scenario
`black-tan-combat-exploration`

## Artifact Path
`logs/browser-smoke/black-tan-combat-exploration/20260604_160539`

## Exact Failure Symptom
The harness selects a `composite` action (e.g., Shield-Trooper's "Multiattack"), but fails to recognize that a sub-action selection is required. It times out waiting for a modal and eventually advances the turn without executing any attacks.

## Reproduction Command
`env INIT_TRACKER_DEBUGGING=1 .venv/bin/python scripts/validation/browser-smoke-harness.py --scenario black-tan-combat-exploration --max-rounds 1 --max-turns 40 --artifact-root logs/browser-smoke --base-url http://127.0.0.1:8787`

## Observed Behavior
At Turn 5, the Shield-Trooper selects "Multiattack". The `/dmcontrol` surface enters a "Resolution Tray" state for selecting multiattack steps, but the harness only checks for `aoeState` and `targetPreviewMode`. It doesn't see either, so it waits for a modal, which never appears because a sub-action hasn't been chosen.

## Expected Behavior
The harness should detect when a `composite` action is active (perhaps via a new `window.__dmcontrolSmoke.compositeState()` hook) and select a sub-action from the tray.

## Relevant Event Log Excerpts
(Event log not written due to crash in subsequent turn)

## Suspected Area
`scripts/validation/browser-smoke-harness.py` and `assets/web/dmcontrol/index.html` (missing hooks for composite/sequence state).

## Recommended Next Bounded Task
Implement `compositeState` hooks in `dmcontrol` and update the harness to handle choosing sub-actions when a composite action is selected.
