## 2. Classification of Models

### A. Pending Reaction Opportunity (Event-Triggered)
*Capabilities that wait for a specific game event and offer a choice.*
*   `interpose-shield` (Shield Trooper)
*   `keep-officer-breathing` (Field Medic)
*   `officer-priority` (Lieutenant - triggered by state change)

### B. Explicit Commander Action (Turn-Triggered)
*Capabilities activated manually on the commander's turn to grant allies opportunities.*
*   `direct-fire` (Lieutenant)
*   `condemn-target` (Captain)
*   `coordinated-volley` (Captain)
*   `make-an-example` (Major)
*   `command-fire` (Major)

### C. One-Shot Modifier Prompt (Resolution-Triggered)
*Capabilities that modify a pending roll or result.*
*   `get-down` (Lieutenant)
*   `not-yet` (Captain)
*   `countermand` (Major)
*   `duck-behind-them` (Major)

### D. Passive/Aura Reminder
*Capabilities that provide constant bonuses or situational triggers without a reaction cost.*
*   `protected-by-rank` (Major)
*   `command-presence` (Lieutenant)
*   `occupation-commander` (Major)

## 3. Proposed Backend Prompt Contract

The backend will maintain a `_pending_prompts` queue. Prompts are generated by resolution logic (attacks, saves) and cleared by DM resolution or expiry.

```json
{
  "prompt_id": "p_12345",
  "source_cid": 1, 
  "source_capability_id": "interpose-shield",
  "trigger_event": "hit",
  "actor_name": "Shield Trooper 1",
  "context_text": "Grunt 2 was hit by Hero (12 damage).",
  "requirements": {
    "reaction_available": true,
    "uses": null,
    "proximity_check": "5ft"
  },
  "options": [
    {
      "label": "Interpose Shield",
      "action": "execute",
      "preview": "Reduce damage to 2.",
      "payload": {"damage_reduction": 10, "consume_reaction": true}
    },
    {
      "label": "Skip",
      "action": "skip",
      "preview": "Take full damage."
    }
  ],
  "expires": "immediate"
}
```

## 4. Proposed /dmcontrol UX Contract

1.  **Overlay Notification:** A non-intrusive card appearing in the top-center or sidebar of the `/dmcontrol` surface.
2.  **Visual Cue:** If the trigger is related to a specific combatant, highlight that combatant in the tracker.
3.  **One-Click Resolution:**
    *   **[Execute]:** Sends the resolution command to the backend. The backend applies effects, logs the result, and consumes the reaction/use.
    *   **[Skip]:** Dismisses the prompt.
4.  **Logging:** Every resolution (even a Skip) results in a combat log entry to ensure transparency.
    *   *Log:* "Shield Trooper 1 uses Interpose Shield to protect Grunt 2 (Reduced 10 damage)."
    *   *Log:* "Major 1 skips Countermand for Rifleman 3."

## 5. Recommended Gate 3B Implementation: Save Modifier Prompts

**Target Capabilities:** `not-yet` (Captain) and `countermand` (Major).
**Rationale:** These share a common "Save Failed -> Prompt -> Reroll/Bonus" flow. 
*   `not-yet` is a self-reaction (lowest complexity).
*   `countermand` is an ally-reaction (adds target resolution).

Both can be implemented by hooking into the existing save resolution loop.
