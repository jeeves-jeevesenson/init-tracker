This document maps every data domain in `init-tracker` to its authoritative runtime source, persistent source, derived caches, and transport lifecycles. It defines the exact boundaries between active state and read projections to prevent future regressions.

## Core Invariants

1. **The Projection Decoupling Invariant:**
   The read-only caches `_cached_snapshot` and `_cached_static_payload` must never store custom runtime attributes that cannot be rebuilt from `InitiativeTracker` objects or `player_profiles`.

2. **The First-Load Hydration Obligation:**
   When a user connects, the static payload handshake must include all relevant player capabilities (spellbooks, resource definitions, inventories, class traits) for seeded characters to ensure immediate UI usability.

3. **The Non-Clobber Delta Rule:**
   High-frequency dynamic updates must never omit or clobber existing player capability states with empty structures (`{}` or `[]`). The frontend must filter empty updates out, and the backend must carry over rich static data when generating partial snapshots.
